## Ф1 — Правка кода проекта в форме (без админки) — консистентность форм

**Хочу:** менять `Project.code` прямо в форме проекта, не заходя в Django-админку.

**Диагноз (консистентность):** `code` в форме уже правят **Изделие**
([engine.py:2655](../backend/plume/engine.py#L2655), `update_item`) и **Склад**
([engine.py:332](../backend/plume/engine.py#L332), `update_location`) — оба с одним
guard'ом: непустой + уникальный-кроме-себя, дружелюбное сообщение. **Проект —
единственная выпадающая форма**: в `update_project` (~2709) `code` намеренно выключен
(«Код и статус здесь не трогаем»), в UI поле read-only. Ф1 = **привести Проект к
общему паттерну**, не изобретать новый.

**Решение (Иван):** править код можно **всем** проектам (в т.ч. внутренним складам,
в любом статусе) — как ведёт себя Изделие. `code` — не PK (pk = `id`), FK ссылаются по
id → переименование **не рвёт** связи/историю; меняется только человекочитаемый ярлык
(`project_code` в ответах engine вычисляется на лету). Стеречь ровно две ошибки:
**пустой код** и **коллизия** — обе дружелюбным предупреждением до сохранения.

**Сделать (дословно паттерн `update_item`):**
- В `engine.update_project` добавить ветку `'code' in changes`: `.strip()`; пустой →
  `ValidationError('Нужен код проекта.')`; `Project.objects.filter(code=code).exclude(
  pk=project.pk).exists()` → `'Проект с кодом X уже есть.'`; иначе `project.code = code`,
  `fields.append('code')`. Убрать из docstring «код … здесь не трогаем».
- HTTP: добавить `'code'` в разрешённые поля `PATCH` в
  [views.py:170](../backend/plume/views.py#L170) (`project_detail`).
- React: снять read-only с поля кода в форме проекта, вписать в §6-замок (правится при
  снятом замке, как name/budget).

**Проверка на консистентность (стандарт Ивана — см. память):** после правки все три
формы (Изделие/Склад/Проект) правят `code` **одинаково**. Заодно свериться, что тексты
guard'ов единообразны и что фронт везде показывает ошибку одинаково.

## Ф0 — ✅ ЗАКРЫТО: стек форм проекта (дубль React-ключа)

**Симптом:** при выборе проекта старая форма не стиралась — новая рендерилась под ней
(стек), затем и изделие липло снизу. Воспроизводилось на проде в инкогнито.

**Причина:** соседи фрагмента `DeficitView` и `ClosurePanel` имели одинаковый
`key={sel.id}` → React «терял» старый `DeficitView` при смене проекта и не размонтировал.

**Сделано:** развёл ключи (`deficit-${sel.id}` / `closure-${sel.id}`) в
[App.tsx](../frontend/src/App.tsx). Пересборка + `collectstatic` + редеплой. Подробности
и урок про ложную диагностику — [JOURNAL.md](JOURNAL.md) (2026-07-10).
