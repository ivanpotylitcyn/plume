# Волна 15 — план реализации

Живой план-чеклист пятнадцатой волны. Источник «что делаем сейчас и в каком
порядке». Значимые решения по ходу — в [JOURNAL.md](JOURNAL.md) (запись

**Цель волны 15:** справочник `Item` в Plume синхронизирован с внешней
**библиотекой компонентов Altium** (внешний git-репозиторий) — она источник правды по
покупным изделиям. Плюс — разморозка модели: поля `Item` подгоняются под канон
библиотеки, оси «что за изделие» и «производим/покупаем» разведены, стоимость
производимого считается роллапом по BOM.

## Объём

**Входит:**
- Переименования полей `Item` под канон библиотеки (полный охват: модель + БД +
  API + фронт) + новая справочная сущность `Category` + новое поле `temperature`
  + смена оси `is_manufactured→produced` + удаление хранимого `active`
  (→ вычисляемое).
- Форма импорта CSV в режиме «Изделия»: мульти-файл → серверный движок парса
  (CP1251) → диф-вью с чекбоксами → применение подтверждённого (полная сверка:
  добавить / обновить / удалить-или-осиротить).
- Рекурсивный роллап `estimated_cost` по BOM + кнопка «Пересчитать стоимость» в
  форме производимого изделия.

**НЕ входит:** RFQ/КП-сущности; парсинг числовых параметров из `Description`
(тащим как есть, текстом); импорт CAD-полей (`Comment`, `Footprint/Library`); свой
инструмент/скрипт с доступами (отвергнут — путь Б).

## Источник правды по библиотеке

Репозиторий библиотеки (берётся локально). CSV в `csv/*.csv`, **CP1251**,
разделитель `;` (без экранирования), LF, финальный перевод строки. 8 колонок
(схема одинакова у всех таблиц, набор таблиц растёт):

| Колонка | Смысл | → Plume |
|---------|-------|---------|
| `Design Item Id` | уникальный ключ Altium = **заказной** Part Number (с суффиксом упаковки) | `Item.design_item_id` (ключ сопоставления) |
| `Comment` | PN без суффикса, печать на схеме | — (не тащим) |
| `Description` | рус. описание | `Item.description` |
| `Footprint/Library Path/Ref` | CAD-ссылки (часто пустые) | — (не тащим) |
| `Temperature` | предельная T компонента, напр. `-40-125°C` | `Item.temperature` |
| *(имя файла)* | класс компонента: `capacitors`/`mcu`/`regulators`/`sensors`/`interfaces` | `Item.category` (→ `Category`) |

Цену библиотека **не хранит принципиально** — `estimated_cost` остаётся за Plume.

## Решения (утверждены на планировании 2026-07-12)

- **Путь Б** — форма в аппе, не внешний скрипт (юзер уже залогинен, 0 доп.
  инструментов в синхроне).
- **Синк = полная сверка**, каждая ветка под ручной галочкой; scoping «пропавших»
  **по категории** (мульти-файл = вся библиотека за раз).
- **Переименования `Item`** (охват полный, без alias-слоя):
  `code→design_item_id` («Изделие»), `name→description` («Описание»),
  `kind→category` (FK → `Category`, «Категория»). Имя `design_item_id` выбрано
  осознанно вместо `item_id`: `item_id` в рукописном JSON-API уже занят как Django
  FK-PK аксессор (`lot.item_id`) и ключ payload'ов мутаций — коллизия имён.
  `design_item_id` — дословный канон библиотеки (`Design Item Id`), конфликта нет.
  См. JOURNAL 2026-07-12 «Грабли имени `item_id`». verbose_name поля — «Изделие».
  **PK-ссылки (`item_id`/`target_item_id`/`component_id`) при этом НЕ трогаем** —
  они остаются PK и в сериализации, и в payload'ах мутаций.
- **`is_manufactured→produced`** (boolean, «Производимое»; ось ⟂ `category`).
- **`active` — убрано, вычисляемое** («используется/спящий» через `Exists` по
  живым ссылкам). Ветка «пропал»: не используется → удалить; используется →
  флаг «сирота, нет в библиотеке».
- **`kind` = справочник `Category` (FK)**, а не enum/CharField: `code` (стем CSV),
  `label` рус., `icon` Codicon; синк `get_or_create` на лету.
- **Миграции пересобираем в чистый `0001_initial`** (dev + прод). Боевых данных
  нет нигде. **Прод не сеем** — Иван заполнит сам. Dev для дебага сеять можно, но
  **прибирать за собой** (синтетические тестовые названия путают). Иван позже даст
  свой тестовый набор — захардкодим в репозиторий.
- **Стартовые `Category` — только из библиотеки** (5 файлов). Прочие классы
  (прибор/крепёж/деталь/узел…) Иван добавит руками — **не выдумывать**.
- **Роллап стоимости — рекурсивный до листьев.** Сценарий: сменили цену резистора
  → кнопка у прибора → обновилась и стоимость платы (в неё входит резистор), и
  прибора (в него входит плата). Промежуточный производимый узел без собственной
  оценки = сумма состава. Ручная наценка — редко, только перед поставкой; оценка —
  частый инструмент планирования/ведения проекта.
