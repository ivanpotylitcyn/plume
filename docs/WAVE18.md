# Волна 18 — импорт BOM из Altium (перенос Ф2 волны 16)

Живой план-чеклист восемнадцатой волны. **Одна тема:** загрузка состава изделия
(BOM), напрямую экспортированного из Altium Designer, с сопоставлением строк по
`design_item_id` с уже синхронизированными из библиотеки (волна 15) изделиями.

**Статус: ЗАПЛАНИРОВАНА, НЕ СТАРТОВАЛА.** Перенесена из [волны 16](WAVE16.md) (была
Ф2). Причина переноса: **ждала формата и образца BOM-файла** от Ивана — без образца
парсер жёстко не проектируем. Стартуем, когда образец на руках.

## Объём

### Кнопка «Загрузить BOM» в форме редактирования изделия
В форме изделия ([ItemView.tsx](../frontend/src/ItemView.tsx), блок «Состав (BOM)»,
для `composable`/производимого) — кнопка **«Загрузить BOM»**. По ней грузим BOM,
экспортированный из Altium Designer. Компоненты BOM — изделия из библиотеки, уже
синхронизированные CSV-импортом волны 15 → сопоставление строк с `Item` по
**`design_item_id`** (заказной PN, ключ библиотеки; та же ось, что в синке В15).

- Переиспользовать наработки В15: серверный парсер + диф/применение, форма загрузки
  (`uploadMulti`/`upload` в [api.ts](../frontend/src/api.ts)), паттерн диф-вью
  (`LibraryImportView`). BOM-импорт логически близок: строки → сопоставление с `Item`
  по `design_item_id` → создание/сверка `BomLine` у текущего изделия.

## Открытые развилки (снять с Иваном при старте — нужен образец файла)
- **Формат файла** — xlsx или csv; кодировка, разделитель, имена/порядок колонок
  (`Design Item Id` + количество? уровни вложенности? refdes?).
- **Плоский BOM** (один уровень, прямые компоненты текущего изделия) **или
  иерархический** (несколько уровней сразу)?
- **Поведение при отсутствии компонента в справочнике** (строка BOM ссылается на
  `design_item_id`, которого нет как `Item`): ошибка / пропуск / предложить
  синхронизировать библиотеку сперва?
- **Замена состава целиком vs слияние** (диф-вью с галочками, как в В15)?
- **Количество/позиция** (`BomLine.qty`/`position`) — откуда в экспорте Altium.

## Пересечения с волной 17 (учесть)
- Волна 17 добавляет `Item.status {draft⇄posted}` = фиксация (как StockDocument).
  Импорт BOM правит `BomLine` → **гейт по статусу цели:** нельзя лить BOM в posted-
  (зафиксированное) изделие без расфиксации. Диф-применение должно уважать замок
  изделия-владельца (или предлагать расфиксировать).
- Импортированные из библиотеки компоненты BOM — уже `posted` (синк волны 17).
